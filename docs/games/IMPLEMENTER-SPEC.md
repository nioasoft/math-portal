## Architecture facts (verified ‚Äî do not deviate)
- Game contract `src/lib/games3d/types.ts`: `Game3D { meta, init(ctx): GameInstance }`. `GameMeta.topic` MUST be one of: `'geometry'|'arithmetic'|'fractions'|'percentage'|'decimals'|'ratio'|'series'|'units'|'wordProblems'|'misc'`. (No 'algebra'/'time'/'money'/'data' ‚Äî map: place-value/number-sense/add-sub/algebra/money‚Üí`arithmetic`; time/measurement‚Üí`units`; data/patterns‚Üí`misc`; pick the closest.)
- `GameInstance`: `{ onFrame?, onResize?, onPause?, onResume?, onQualityDowngrade?, dispose() }`.
- `SceneContext` (`src/lib/games3d/engine/SceneContext.ts`): `ctx.prompt.set/clear`, `ctx.controls.set/clear`, `ctx.status.set/clear`, `ctx.score.add/set/get`, `ctx.feedback.correct/wrong/hint`, `ctx.audio.play('success'|'fail'|'click')`, `ctx.input.on('tap'|'dragStart'|'drag'|'dragEnd', handler)‚ÜíUnsubscribe`, `ctx.presets.camera.orbit(target,dist)/topDown(target,dist)/locked(pos,lookAt)`, `ctx.presets.lighting.daylight/soft/dramatic`, `ctx.t(key, params?)`, `ctx.complete(summary)`, `ctx.mode` ('practice'|'quiz'), `ctx.prefersReducedMotion`, `ctx.scene/camera/renderer`.
- `PointerInfo`: `{ x, y, pixelX, pixelY, picked }`. `x`,`y` are NDC in ‚àí1..1; `picked` is the `THREE.Object3D` under the pointer (use for tap/drag-to-pick). Ensure your camera framing makes NDC‚Üíworld monotonic (no mirrored axis) so drag directions are NATURAL (right=more/wider, up=more/taller; never inverted).
- `ControlButton`: `{ id, label, onPress, variant?: 'default'|'confirm'|'reset' }`. Rendered as solid, accessible HTML buttons by the overlay (so they always have contrast).
- `GameStatus`: `{ stars?, maxStars?, streak?, progress?:{current,total} }`.
- Quiz: `import { createQuizController } from '@/lib/games3d/quiz/controller'`. `createQuizController(generator, { length: 10, pointsPerCorrect: 10 })`. `.state()`‚Üí`{ current, index, total, score, correct, streak, finished }`; `.submit(answer)`‚Üí`{ correct, finished }`; `.summary()`‚Üí`{ totalPoints, accuracy, durationSec, streak }`. Generator implements `ProblemGenerator<TProblem>` from `@/lib/games3d/quiz/types`: `next(): TProblem` + `check(problem, answer: unknown): boolean`.
- **Per-face materials gotcha:** `roundedBox()` returns a `RoundedBoxGeometry` which has only ONE material group ‚Äî passing a per-face material ARRAY to it is SILENTLY IGNORED (only `material[0]` renders on the whole mesh). To put a different material/texture on a specific face (e.g. a number/decal on a cube face), use a plain `THREE.BoxGeometry` (which has 6 groups in order `0:+x, 1:‚àíx, 2:+y top, 3:‚àíy, 4:+z FRONT, 5:‚àíz`) and put the decal on the **+z front face (index 4)** so it faces the default front camera, OR attach a separate textured plane as a child mesh. Don't assume `roundedBox` honors a material array.
- Kit: `import { applyClayLook, roundedBox, popIn, punch, shake, celebrate, bigCelebrate, computeStars, PALETTE } from '@/lib/games3d/kit'`. `applyClayLook(ctx, {topColor, bottomColor, ground, shadowArea, fog})`‚Üí`{dispose}`. `roundedBox(w,h,d,radius,segments)`. `popIn(obj,{delay?,scale?})`, `punch(obj, amount)`, `shake(obj, amount, ms)`, `tweenTo(from,to,ms,onUpdate)` (from `@/lib/games3d/kit/juice` if needed; returns a Tween). `computeStars(accuracy)`‚Üí0..3. `PALETTE` has named colors (mint, coral, sky, grape, sun, cream, ‚Ä¶). The ENGINE auto-updates the shared tween group every frame ‚Äî just create tweens via the kit and TRACK them; do NOT add your own RAF.

## THE DEFINITIONS ‚Äî a game is NOT done until ALL hold (reviewer + a human browser check enforce these)
1. Folder `src/lib/games3d/games/<id>/` with `problems.ts` (+ `__tests__/problems.test.ts`) and `<PascalName>Game.ts` exporting `export const <camelName>Game: Game3D`.
2. Registered in the two files above. Nothing else (no stray files/routes).
3. i18n block in all 6 locales under `games3d`, merged. `meta.i18nKey = "games3d.<suffix>"`.
4. `meta`: correct `id` (kebab, == loader key), `i18nKey`, valid `topic`, `gradeRange` (from catalog), `difficulty` (1‚Äì5), `estimatedSeconds`, `supportedModes: ['practice','quiz']`.
5. BOTH modes: **practice** (endless, NO fail-out ‚Äî a wrong answer KEEPS the built state so the child can fix it) AND **quiz** (10 problems, scored; on finish: `computeStars(accuracy)` + `bigCelebrate()` then `ctx.complete(summary)`).
6. A wrong answer must be GENUINELY POSSIBLE ‚Äî the child commits via the **Check** button (no auto-win when state matches). Quiz accuracy must be able to be < 100%.
7. Quiz: after `quiz.submit(answer)` call `ctx.score.set(quiz.state().score)`. Track a `streak`; set `ctx.status.set({ streak, progress:{ current, total:10 } })` (quiz) or `{ streak, stars, maxStars:3 }` (practice).
8. Controls via `ctx.controls.set([...])`: rich pointer interaction PLUS **Check** (`variant:'confirm'`) and **Reset** (`variant:'reset'`). Labels via `ctx.t('controls.*')`. Re-`set` controls on state change OR use a live `state` object the closures read. The exact pointer vocabulary depends on the answer TYPE:
   - **Scalar-answer games** (count/length/height/value the child dials in): provide BOTH on-screen **‚àí/+** buttons (fine control/undo) AND fast **drag** for building/sizing. This is the common case.
   - **Spatial/toggle games** (paint/toggle grid cells, place or drag objects to positions, stretch bands, sort into bins, fold): the natural control IS **tap and/or drag** on the canvas (tap a cell to toggle, drag to paint/move/stretch) + **Check** + **Reset**. Here ‚àí/+ buttons are NOT required and forcing them is an anti-pattern ‚Äî there is no scalar to increment. Reviewers MUST NOT fail a spatial/toggle game for lacking ‚àí/+; instead verify the tap/drag interaction is present, natural, and unsubscribed in dispose. Note: the `InputAdapter` emits `'tap'` only on real canvas PointerEvents (pointerdown‚Üípointerup within move/hold thresholds); JS-dispatched PointerEvents work but synthetic mouse `click`s do NOT trigger `'tap'`.
9. **Natural input directions** ‚Äî drag up = more, right = wider, clockwise = forward, etc. NEVER inverted. Mind Three's conventions (`CylinderGeometry` angle Œ∏‚Üí(x,z)=(sinŒ∏,cosŒ∏); clock 12 at top ‚Üí angle 2œÄ¬∑v/N from +Y, `rotation.z = ‚àíŒ∏`).
10. Theme each game with its catalog `theme` as a real-world PROCEDURAL skin (geometry only, NO asset files). Call `applyClayLook(ctx, ‚Ä¶)` and dispose it.
11. **Prompt shows the TASK ONLY** via `ctx.prompt.set(...)` ‚Äî the target + nothing that reveals success. Do NOT echo the live built value in the prompt; the 3D scene shows progress. The `instructions` i18n string (shown under the prompt) must accurately describe the real interaction. Success is revealed ONLY after Check (the feedback toast).
12. **Contrast:** every overlay text element sits on a dark pill or solid button ‚Äî NEVER light text on the light clay background. In-scene labels use dark/saturated materials, not faint light-on-light. (Control buttons are already solid.)
12b. **Framing (avoid empty/blank/tiny scenes):** the main elements must FILL the central viewport like the sibling games ‚Äî not float as tiny specks with lots of empty space, and never blank. Center your content near the origin and size the camera distance to fit the content's largest extent with a little margin (a wide/flat scene is bound by its WIDTH; reason about the 60¬∞ vertical FOV √ó ~1.4 aspect). If any content sits at or below y=0, either pass `ground: false` to `applyClayLook` OR keep the content above the ground ‚Äî otherwise the clay ground plane (a horizontal plane at y‚âà0) OCCLUDES it and you get a blank clay view. A straight-on `locked` camera at `(0,0,D)` looking at the origin (like `clock-builder`) is a safe default for front-facing scenes and keeps drag-x ‚Üí world-x monotonic.
13. Use `ctx.feedback.correct(...)`/`wrong(...)` for transient feedback; `celebrate()` + `punch()` on correct, gentle `shake()` on wrong (never punishing).
13b. **`popIn`/`punch` use UNIFORM `scale.setScalar` ‚Äî never apply them to a mesh whose scale you set NON-uniformly.** If a mesh has e.g. `mesh.scale.set(1, h, 1)` (a bar sized by value, a liquid column, etc.), calling `popIn`/`punch` on THAT mesh will reset all axes to ~1 and DESTROY the height. Instead wrap the mesh in a parent `THREE.Group`, set the non-uniform scale on the inner mesh, and `popIn`/`punch` the PARENT group (uniform scale on the group is harmless). Symptom if you get this wrong: value-proportional shapes render as uniform-size tiles floating at the wrong position.
13c. **`popIn` snaps its target to `scale‚âà0.001` SYNCHRONOUSLY and only grows back on later frames ‚Äî so never `popIn` an object whose visibility you depend on if any code path can `stopAllTweens()` before that tween finishes.** `Tween.stop()` does NOT fire `onComplete`, so a stopped popIn leaves its target stuck at 0.001 = invisible forever. In particular: do NOT call `stopAllTweens()` mid-lifecycle (e.g. inside a re-flash/rebuild) AND `popIn` a static prop during `init()` ‚Äî the stop kills the prop's grow-back. For an always-visible static prop set `obj.scale.setScalar(1)` explicitly instead of `popIn`. Keep `stopAllTweens()` confined to `dispose()` unless you are certain no entrance tween is in flight. Symptom: the entire 3D scene is empty (only the background gradient renders), no console errors.
8c. **Tap-pick games: a decorative opaque panel coplanar with / in front of your pickable meshes will STEAL the tap.** `ctx.input`'s `picked` is the single TOPMOST raycast hit across the whole scene. If a solid board/tray/background panel sits at the same Z as (or in front of) your pickable slots/pieces/cells, the ray hits the panel (which has no `userData.pieceId`/`slotId`/cell) and the placement/toggle silently never fires ‚Äî the game looks rendered but is unplayable. Fix: make decorative panels non-pickable with `panel.raycast = () => {};` (standard Three.js), OR put the pickable meshes clearly in front of the panel (bigger Z gap, not coplanar). Symptom: tap-select works on tray items but tapping the target area does nothing.
14. **`dispose()` leak-free (TOP PRIORITY):** unsubscribe every input listener; `ctx.controls.clear()` + `ctx.prompt.clear()` + `ctx.status.clear()`; `clayLook.dispose()`; stop ALL game tweens (tracked Set + `track()`); clear any `setTimeout`/`setInterval` you created (track them too); remove every object added to the scene; dispose every geometry AND material you created. Reuse shared geo/mat (one per kind); rebuilding (e.g. re-ticking) must reuse a shared geo/mat or a pool ‚Äî never allocate-and-forget per element per update.
15. **Procedural only**, ‚â§ perf budget, target 60fps mobile. Use **Context7 MCP** to confirm any library API before using it (`three` geometries/materials, `@tweenjs/tween.js` v25, `canvas-confetti`) ‚Äî don't guess.
16. **TDD the logic:** write `problems.ts` + generator tests FIRST (valid ranges per grade, NO degenerate/unsolvable problems, STRICT `check()` that rejects off-by-one, wrong types, NaN/null/objects). red‚Üígreen.
